The application is deployed using a modern **Microservices Architecture**:

### 1. **Frontend (Client)**
*   **Technology:** React + Vite
*   **Hosting:** Google Firebase Hosting (CDN)
*   **Configuration:** 
    *   Build optimized for production.
    *   Environment variable `VITE_API_URL` points to the live Render Backend.
    *   Single Page Application (SPA) rewrite rules enabled in `firebase.json`.

### 2. **Backend (Orchestrator)**
*   **Technology:** Node.js + Express
*   **Hosting:** Render
*   **Role:** 
    *   Handles API requests from the frontend.
    *   Manages authentication and database interactions (Firebase Admin SDK).
    *   Acts as a gateway, forwarding image analysis requests to the AI Service.
*   **Security:**
    *   `FIREBASE_SERVICE_ACCOUNT`: Securely injected via Environment Variables.
    *   CORS configured to allow requests from the Frontend.

### 3. **AI Service (Intelligence)**
*   **Technology:** Python + FastAPI + TensorFlow/PyTorch
*   **Hosting:** Render
*   **Role:**
    *   Specialized microservice for Image Processing.
    *   Performs color analysis, texture detection, and disease classification.
    *   **New Feature:** Includes edge-case handling for "Non-Crop" images.

### 4. **Data Layer**
*   **Database:** Cloud Firestore (NoSQL) - Stores user data, post history, and community discussions.
*   **Storage:** Cloud Storage for Firebase - Stores uploaded crop images publicly for report generation.

## ğŸ”„ Data Flow

1.  **User** uploads an image on `smart-doctor-crop.web.app`.
2.  **Frontend** sends the image to `crop-backend-avko.onrender.com/api/detect`.
3.  **Backend** forwards the image to the **AI Service**.
4.  **AI Service** processes the image:
    *   Checks if it is a valid crop.
    *   If valid, detects disease and confidence.
5.  **Backend** saves the record to **Firestore** and the image to **Storage**.
6.  **Frontend** receives the result and displays the "Disease Report" or "No Crop Detected" warning.
